# Project explanation template

Use this template to create a concise, structured explanation for each project.

1. What it is
- One-sentence summary of the project and its scope.

2. Why you built it
- Describe the problem or gap this project addresses and why it mattered.

3. How it works
- High-level architecture and data/interaction flow. Mention key technologies and components.

4. What you learned
- Short, practical lessons: trade-offs, performance work, team/process insights, or measurable impact.

Keep each section concise and factual — suitable for case studies, outreach, and interviews.
