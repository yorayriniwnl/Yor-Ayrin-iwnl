After each completed code change in this repository, create a small descriptive commit and push it to GitHub unless the user explicitly says not to.

If the user types `///` in VS Code chat, treat it as a request to commit and push the current project to GitHub.

Keep commits scoped to the completed task.

Do not silently bundle unrelated uncommitted changes into the same commit.

If commit or push fails because of auth, network, branch protection, merge conflicts, or remote state, report it immediately.
